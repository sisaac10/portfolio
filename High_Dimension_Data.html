PCA¶

The principal component analysis (PCA) is, "A technique used to emphasize variation and bring out strong patterns in a dataset. It's often used to make data easy to explore and visualize." The way that PCA reduces dimensionality is that it,"Finds a new coordinate system in which every point has a new (x,y) value. The axes don't actually mean anything physical; they're combinations of height and weight called "principal components" that are chosen to give one axes lots of variation." ( http://setosa.io/ev/principal-component-analysis/ - "Principal Component Analysis" Victor Powell )

The goal of dimensionality reduction is to reduce the number of variables (dimensions, measurements, columns). For example, in the Iris dataset we have four variables (sepal_length, sepal_width, petal_length, petal_width). If we can reduce the number of variables to two, then we can easily visualize them. PCA offers one way to do this.

PCA is already implemented in the scikit-learn package, a machine learning library in Python, which should have been included in Anaconda. If not, to install scikit-learn, run:

conda install scikit-learn

or

pip install scikit-learn

Before running PCA, we need to transform the iris from DataFrame to Numpy's array object. DataFrame.values returns the Numpy representation of DataFrame.

In [14]:
X = iris.values[:, 0:4] # extract the 1st to the 3rd columns of all rows
Y = iris.values[:, 4]  # extract the 4th column of all rows

We can now perform PCA with the following code:

In [15]:
from sklearn.decomposition import PCA
pca = PCA(n_components=2) # set the number of components to 2
X_r = pca.fit(X).transform(X)

#Make a dataframe with the results
df = pd.DataFrame(X_r, columns=['PC1', 'PC2'])
df['species'] = Y

Now we only have two dimensions. We can plot them again with the previous code:

In [16]:
sns.pairplot(df, hue='species')

t-SNE¶

t-SNE (t-Distributed Stochastic Neighbor Embedding) is also tool used to visualize high-dimensional data. The technique has become widespread in the field of machine learning, since it has an ability to create compelling two-dimensonal maps from data with hundreds or even thousands of dimensions.

In [17]:
from sklearn.manifold import TSNE
from sklearn.datasets import load_iris

iris = load_iris()
X_tsne = TSNE(learning_rate=100, perplexity=30).fit_transform(iris.data)
plt.figure(figsize=(10, 5))
plt.subplot(121)
plt.scatter(X_tsne[:, 0], X_tsne[:, 1], c=iris.target, cmap='Set1', s=30)
Out[17]:
<matplotlib.collections.PathCollection at 0x24c8a812550>

The hyperparameter perplexity determines how to balance attention between local and global aspects of your data. Changing this parameter (default is 30) can cause drastic changes in the output:

Perplexity is a, "Parameter that is, in a sense, a guess about the number of close neighbors each point has. The perplexity value has a complex effect on the resulting pictures. For the algorithm to operate properly, the perplexity really should be smaller than the number of points. Implementations can give unexpected behavior otherwise."( https://distill.pub/2016/misread-tsne/ - Wattenberg, et al., "How to Use t-SNE Effectively", Distill, 2016 )
